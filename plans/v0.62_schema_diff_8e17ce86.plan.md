# v0.62 -- Schema + Diff

## Context

The roadmap calls for two new pure "a" stdlib modules:
- **`std/schema.a`** -- JSON Schema (draft-07 subset) validation
- **`std/diff.a`** -- unified text diff (Myers algorithm), patch application, structured line diff

Both are pure "a" modules with no new C/Rust builtins needed. They follow the established stdlib patterns: `_`-prefixed private helpers, `type_of` guards, `Ok`/`Err` returns, `str.chars` loops, `str.lines` for text splitting.

**Key constraint:** `regex.*` builtins exist only in the Rust VM -- they are NOT wired in cgen or the C runtime. The schema `pattern` keyword will use `regex.is_match` with a fallback note that it only works in the Rust VM, or we implement a basic pattern matcher. Given `std/re.a` already exists as a pure "a" regex engine, we can use `re.test(pattern, value)` from `std.re` for native-compatible pattern matching.

## Part 1: `std/schema.a` (~200 lines)

Pure "a" module. Validates values against JSON Schema draft-07 subset.

### Public API

```
schema.validate(value, schema) -> Ok(value) | Err("path: message")
schema.from_type(type_str)     -> schema map
```

### Supported keywords

- **Type checking**: `"type"` -- `"string"`, `"number"`, `"integer"`, `"boolean"`, `"array"`, `"object"`, `"null"`
- **String constraints**: `"minLength"`, `"maxLength"`, `"pattern"` (via `use std.re` for native compat)
- **Numeric constraints**: `"minimum"`, `"maximum"`
- **Array constraints**: `"items"` (single schema applied to all elements), `"minItems"`, `"maxItems"`
- **Object constraints**: `"properties"` (map of prop name -> sub-schema), `"required"` (array of required keys)
- **Enum**: `"enum"` (array of allowed values)

### Implementation structure

```
use std.re

fn validate(value, schema) -> result
  ; Entry point -- delegates to _validate_at(value, schema, "")
  ; Returns Ok(value) on success, Err("path: message") on first failure

fn _validate_at(value, schema, path) -> result
  ; Core recursive validator
  ; 1. Check "type" keyword via type_of(value) -> schema type mapping
  ; 2. Check "enum" via loop comparison
  ; 3. Dispatch to _validate_string, _validate_number, _validate_array, _validate_object
  ; 4. Return Ok(value) if all pass

fn _type_matches(value, schema_type) -> bool
  ; Maps type_of() tags to JSON Schema types:
  ;   "str" -> "string", "int" -> "integer"/"number", "float" -> "number"
  ;   "bool" -> "boolean", "array" -> "array", "map" -> "object"
  ;   "void" -> "null"

fn _validate_string(value, schema, path) -> result
  ; minLength, maxLength via len(str.chars(value))
  ; pattern via re.test(schema["pattern"], value)

fn _validate_number(value, schema, path) -> result
  ; minimum, maximum via comparison

fn _validate_array(value, schema, path) -> result
  ; minItems, maxItems via len(value)
  ; items: loop and recursively validate each element at path[i]

fn _validate_object(value, schema, path) -> result
  ; required: check map.has(value, key) for each
  ; properties: loop keys, recursively validate at path.key

fn from_type(type_str) -> map
  ; Converts "a"-style type strings to schema maps:
  ; "str" -> #{"type":"string"}, "int" -> #{"type":"integer"}, etc.
  ; "{name: str, age: int}" -> object schema with properties
```

Key files to reference: [std/yaml.a](std/yaml.a) for parsing patterns, [std/html.a](std/html.a) for recursive tree validation, [std/re.a](std/re.a) for `re.test()`.

## Part 2: `std/diff.a` (~200 lines)

Pure "a" module implementing Myers diff algorithm.

### Public API

```
diff.text(a, b)            -> unified diff string
diff.lines(a, b)           -> [#{"op":"add"|"remove"|"keep", "line":"..."}]
diff.patch(original, diff_str) -> patched string
```

### Implementation structure

```
fn text(a, b) -> str
  ; 1. Split both strings with str.lines()
  ; 2. Run _myers(lines_a, lines_b) to get edit script
  ; 3. Format as unified diff: --- a, +++ b, @@ hunks @@

fn lines(a, b) -> array
  ; 1. Split, run _myers
  ; 2. Return structured array of {op, line} maps

fn _myers(a_lines, b_lines) -> array of {op, line}
  ; Myers shortest edit script algorithm:
  ; - Build V arrays for each D (edit distance 0..N+M)
  ; - Find the shortest path from (0,0) to (N,M)
  ; - Backtrack to recover the edit script
  ; - Returns array of "keep"/"add"/"remove" ops

fn _format_unified(ops, a_lines, b_lines) -> str
  ; Group operations into hunks (with context lines)
  ; Output @@ -start,count +start,count @@ headers
  ; Prefix lines with " " (keep), "+" (add), "-" (remove)

fn patch(original, diff_str) -> str
  ; Parse unified diff hunks from diff_str
  ; Apply additions/removals to original lines
  ; Return reassembled string
```

The Myers algorithm operates on the "shortest edit script" principle: find the minimum number of insertions and deletions to transform sequence A into B, greedily extending diagonal (equal) runs. For the "a" language, this will be implemented using arrays as the V frontier (indexed by diagonal k). The max edit distance is `len(a) + len(b)`, but typical diffs are much smaller.

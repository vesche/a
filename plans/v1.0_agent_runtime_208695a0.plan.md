# v1.0 -- The Agent Runtime

## What's left

The language is functionally complete. v0.57-v0.68 delivered: bootstrap from C, in-process HTTP/TLS, MCP, streaming, agents, schema/diff, image processing, packages, native toolchain, concurrency, and self-improvement. Three cracks remain.

## Crack 1: Remove curl fallback from C runtime

[c_runtime/runtime.c](c_runtime/runtime.c) line ~2527 falls back to `execvp("curl", ...)` when in-process TLS setup fails. This is the last external tool dependency.

- **Action**: Remove `http_request_curl()` entirely. When TLS fails, return an error result instead of silently shelling out. The in-process HTTP+TLS path (macOS Security framework / OpenSSL) already handles the vast majority of cases.
- **Action**: Add a clear error message when TLS connection fails: `"HTTPS connection failed (no TLS available)"` so the caller can diagnose.
- **Action**: Remove the `#include` and function declaration for the curl path.
- **Impact**: ~50 lines removed from runtime.c. The binary no longer needs curl on the host.

## Crack 2: Make gcc-only bootstrap the primary path

[build.sh](build.sh) still uses `cargo run` as Step 1. [bootstrap/build.sh](bootstrap/build.sh) already does pure gcc. The top-level script should default to the gcc path.

- **Action**: Rewrite `build.sh` to:
  1. Compile from `bootstrap/cli.c` + C runtime sources via gcc (the current `bootstrap/build.sh` logic)
  2. Use the resulting `./a` binary to rebuild itself (`./a build src/cli.a -o a2`)
  3. Verify `a2` works (`a2 run examples/hello.a`)
  4. Replace `./a` with `a2`
- **Action**: Move the Rust-based build to `build_rust.sh` (for development/testing only, not required)
- **Action**: Update [README.md](README.md) to reflect gcc-only as the default build

## Crack 3: Native test parity

`cargo test` runs Rust integration tests that the native harness doesn't cover. Grow native test coverage so `./a test tests/native/` is sufficient for release confidence.

- **Action**: Audit [tests/integration_tests.rs](tests/integration_tests.rs) to identify test categories not covered by native tests
- **Action**: Add native test files for uncovered areas (likely: edge cases in lexer/parser, string interpolation, destructuring, pattern matching, error recovery, pipe operator, closures over mutable state, module resolution)
- **Action**: Add a `tests/native/test_self_host.a` that verifies `./a build src/cli.a` produces a working binary (the ultimate regression test)
- **Target**: Native tests should cover every major language feature and runtime capability

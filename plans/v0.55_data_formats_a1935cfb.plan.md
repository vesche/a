# v0.55 -- Data Formats

All four modules are pure "a" implementations -- no C runtime changes, no Rust VM changes needed. New files go in `std/`, loaded via `use std.yaml` etc.

## 1. `std/yaml.a` -- YAML 1.2 subset

Parse and stringify the YAML subset agents encounter (config files, CI definitions, k8s manifests):

- **Mappings**: `key: value`, nested via indentation
- **Sequences**: `- item` lists, nested
- **Scalars**: plain strings, `'single-quoted'`, `"double-quoted"`, integers, floats, `true`/`false`/`null`
- **Multiline**: `|` (literal block), `>` (folded block)
- **Comments**: `# ...` stripped

Functions: `parse(s) -> map/array`, `stringify(v) -> str`

Strategy: line-by-line parser tracking indent level. Each line is classified (mapping, sequence item, block scalar indicator, comment, blank). Indent changes drive push/pop of a context stack.

## 2. `std/toml.a` -- TOML

Parse and stringify TOML (Cargo.toml, pyproject.toml, config files):

- **Key-value pairs**: `key = value`
- **Tables**: `[section]`, `[section.subsection]`
- **Array of tables**: `[[section]]`
- **Values**: strings (basic `"..."`, literal `'...'`), integers, floats, booleans, arrays `[...]`, inline tables `{...}`
- **Comments**: `# ...`

Functions: `parse(s) -> map`, `stringify(v) -> str`

Strategy: line-by-line parser. Lines starting with `[` switch the current table context. Key-value lines split on first `=`, then parse the value (string literals, numbers, booleans, arrays, inline tables).

## 3. `std/html.a` -- HTML parser + CSS selector

Parse HTML into a DOM-like tree of tagged maps, then query with CSS selectors:

- **Tags**: open `<tag>`, close `</tag>`, self-closing `<tag/>`
- **Attributes**: `<tag attr="value" flag>`
- **Text nodes**: content between tags
- **Nesting**: builds a tree

DOM node structure:
```
#{ "tag": "div", "attrs": #{ "class": "main" }, "children": [...] }
#{ "tag": "#text", "text": "hello" }
```

CSS selectors (subset): `tag`, `.class`, `#id`, `tag.class`, descendant (space-separated), comma groups.

Functions: `parse(s) -> node`, `select(node, selector) -> [node]`, `text(node) -> str`

Strategy: character-by-character lexer that emits open-tag, close-tag, self-closing-tag, and text tokens. A stack-based builder assembles the tree. Selector engine parses the selector string into parts and recursively matches against the tree.

## 4. `std/url.a` -- URL parsing

Extend beyond `encoding.a`'s `url_encode`/`url_decode` with full URL structure parsing:

Functions: `parse(s) -> map`, `encode(s) -> str`, `decode(s) -> str`, `build(parts) -> str`

`parse("https://user:pass@host:8080/path?q=1&r=2#frag")` returns:
```
#{ "scheme": "https", "user": "user", "password": "pass",
   "host": "host", "port": 8080, "path": "/path",
   "query": "q=1&r=2", "params": #{"q":"1","r":"2"}, "fragment": "frag" }
```

`encode`/`decode` delegate to the existing helpers in `std/encoding.a` (imported via `use std.encoding`).

Strategy: sequential scan for `://`, then `@`, then `:port`, then `?`, then `#`. Query string split on `&` then `=`.

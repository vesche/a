# v0.68 -- Self-Improvement Loop

## Context

The language already has the building blocks for self-modification:
- `std/meta.a` -- AST walk/transform/parse/emit pipeline
- `std/compiler/parser.a` -- self-hosted parser returning tagged-map ASTs
- `std/compiler/emitter.a` -- AST-to-source pretty-printer
- `std/compiler/checker.a` -- static analysis (scope, arity, unused vars)
- `std/compiler/cgen.a` -- C codegen from AST
- `std/llm.a` -- unified LLM client (OpenAI/Anthropic/Google)

Neither `std/codegen.a` nor `std/refactor.a` exist yet. No sandbox execution exists.

## New Module: `std/codegen.a`

Uses: `std.compiler.parser`, `std.compiler.checker`, `std.compiler.emitter`, `std.meta`, `std.llm`

### `codegen.compile_check(source) -> map`

Parse and statically check source without executing. Returns:
- `Ok(#{ "ast": ast, "warnings": [...] })` on success
- `Err(#{ "errors": [...], "warnings": [...] })` on failure

Implementation:
1. `parser.parse(source)` -- check for `ParseError`
2. `checker.check(ast)` -- collect diagnostics
3. Split diagnostics into errors vs warnings
4. Return Ok/Err result

### `codegen.run_in_sandbox(source, opts) -> map`

Compile and execute with timeout and output capture. Returns `#{ "stdout": str, "stderr": str, "code": int, "ok": bool }`.

Implementation:
1. Write source to temp `.a` file
2. Shell out to `argv0() + " run " + tmp_path` via `exec()`, wrapped with a timeout
3. Actually, use `timeout(ms, fn() => exec(...))` builtin for enforcement
4. Capture stdout, stderr, exit code from exec result
5. Clean up temp file

The "sandbox" is practical isolation: temp directory, configurable timeout (default 5s via `opts["timeout_ms"]`), captured output. Full OS-level sandboxing (seccomp, etc.) is post-v1.0.

### `codegen.test(source, test_cases) -> map`

Run source against test cases. Each test case is `#{ "input": str, "expected": str }` where `input` is passed as argv and `expected` is matched against stdout.

Implementation:
1. `compile_check(source)` -- fail fast on parse/check errors
2. Write source to temp file
3. For each test case: run via `exec(argv0() + " run " + tmp + " " + test_case["input"])`, compare trimmed stdout to expected
4. Return `#{ "passed": int, "failed": int, "results": [...] }`

### `codegen.generate(description, context, opts) -> map`

Use an LLM to generate "a" code from a natural language description.

Implementation:
1. Build a system prompt describing "a" language syntax, conventions, and available builtins
2. Include `context` info: `context["functions"]` (available fn signatures), `context["types"]`, `context["imports"]`
3. Call `llm.chat(provider, model, messages, opts)` where provider/model come from `opts` (defaults: `"anthropic"`, `"claude-sonnet-4-20250514"`)
4. Extract code from the response (strip markdown fences if present)
5. Run `compile_check` on the generated code
6. Return `Ok(#{ "source": str, "ast": ast })` or `Err(#{ "source": str, "errors": [...] })`

## New Module: `std/refactor.a`

Uses: `std.meta`, `std.compiler.parser`, `std.compiler.emitter`, `std.compiler.ast`

All functions take source string, return transformed source string.

### `refactor.rename(source, old_name, new_name) -> str`

Rename a symbol (function or variable) across a source file.

Implementation:
1. Parse source
2. `meta.walk` the AST, collecting all `Ident` nodes where `name == old_name`
3. Also update `FnDecl` names, `Let` binding names, `Param` names, `Call` targets, `FieldAccess` identifiers
4. Rebuild AST with replacements using `meta.transform`
5. Emit back to source

### `refactor.extract_fn(source, start_line, end_line, fn_name) -> str`

Extract a range of lines into a new function.

Implementation:
1. Parse source
2. Use line-based approach: split source into lines, extract `start_line..end_line`
3. Scan extracted lines for free variables (identifiers used but not defined locally) -- these become parameters
4. Build new function AST node via `meta.gen_fn(fn_name, params, body_stmts)`
5. Replace original lines with a call to the new function
6. Use `meta.add_items` to add the new function to the program
7. Emit

### `refactor.inline_fn(source, fn_name) -> str`

Inline a function at its call sites.

Implementation:
1. Parse source, find the `FnDecl` with matching name via `meta.find_fns`
2. Find all `Call` nodes targeting that function via `meta.find_calls` + `meta.extract_call_name`
3. For single-expression bodies: replace call with the body expression, substituting params with args
4. For block bodies: replace call site with a block expression
5. Remove the original function definition via `meta.filter_items`
6. Emit

## Example: `examples/self_extend.a`

A demonstration script that:
1. Reads its own source via `io.read_file(argv0_source())` or a target file
2. Analyzes it with `meta.fn_signatures` / `meta.call_graph` to understand structure
3. Identifies a "missing capability" (e.g., a TODO comment or a stub function)
4. Uses `codegen.generate` to produce the implementation
5. Runs `codegen.compile_check` and `codegen.test` to validate
6. Writes the improved source back

Requires an API key to run (documented in file header). Designed to be read as a reference, not necessarily executed in CI.

## Test Suite

### `tests/native/test_codegen.a`

Tests for `compile_check`, `run_in_sandbox`, and `test` (the non-LLM parts). No API key needed:
- `compile_check` on valid source returns Ok with AST
- `compile_check` on invalid source returns Err with errors
- `run_in_sandbox` on `println(42)` returns stdout "42", code 0
- `run_in_sandbox` with timeout on infinite loop returns non-zero code
- `codegen.test` with simple test cases passes/fails correctly

### `tests/native/test_refactor.a`

- Rename variable in simple program
- Rename function name
- Extract function from lines
- Inline single-expression function

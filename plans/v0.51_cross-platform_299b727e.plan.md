# v0.51 -- Cross-Platform

## Problem

Three places hardcode the macOS-only linker flag `-Wl,-stack_size,0x10000000`:
- [build.sh](build.sh) lines 30 and 40
- [src/cli.a](src/cli.a) `_gcc_flags()` at line 36
- [scripts/test_memory.sh](scripts/test_memory.sh) line 13

Linux `ld` uses different syntax: `-Wl,-z,stacksize=268435456`. The HTTP builtins in the C runtime already use `fork`+`execvp("curl")` which is POSIX-portable, but `curl` must be available on the system.

## Changes

### 1. Platform detection in `_gcc_flags()` -- [src/cli.a](src/cli.a)

Use `exec("uname -s")` to detect the OS at runtime:

```
fn _gcc_flags() -> str {
  let uname_result = exec("uname -s")
  let os = str.trim(uname_result["stdout"])
  if os == "Darwin" {
    ret "-lm -O2 -Wl,-stack_size,0x10000000"
  }
  ret "-lm -O2 -Wl,-z,stacksize=268435456"
}
```

This covers macOS (Darwin) and Linux transparently. Any future POSIX platform would use the Linux-style flags as default.

### 2. Platform detection in `build.sh` -- [build.sh](build.sh)

Add a helper at the top that selects the right flags based on `uname -s`:

```bash
OS=$(uname -s)
if [ "$OS" = "Darwin" ]; then
    STACK_FLAGS="-Wl,-stack_size,0x10000000"
else
    STACK_FLAGS="-Wl,-z,stacksize=268435456"
fi
```

Replace the two hardcoded `-Wl,-stack_size,0x10000000` on lines 30 and 40 with `$STACK_FLAGS`.

### 3. Platform detection in `scripts/test_memory.sh` -- [scripts/test_memory.sh](scripts/test_memory.sh)

Same pattern: detect `uname -s` and set `SFLAGS` accordingly (line 13).

### 4. GitHub Actions CI -- `.github/workflows/ci.yml` (new file)

Matrix build on `ubuntu-latest` and `macos-latest`:

- Install dependencies (`gcc`, `curl` on Ubuntu -- both usually present, but explicit is safer)
- Run `cargo test` (Rust VM tests)
- Run `./build.sh` (native bootstrap)
- Run `./a run examples/hello.a` (smoke test the native CLI)
- Run `./a test tests/native/` (native test suite)

### 5. GitHub Actions release workflow -- `.github/workflows/release.yml` (new file)

Triggered on version tags (`v*`). Matrix: `ubuntu-latest` (x86_64), `ubuntu-latest` + cross-compile (arm64), `macos-latest` (arm64). Each job:

- Builds `./a` and `./a-lsp` via `build.sh`
- Tarballs the binaries: `a-{version}-{os}-{arch}.tar.gz`
- Creates a GitHub Release with the three tarballs attached

For linux-arm64, use `aarch64-linux-gnu-gcc` as the cross-compiler (available via `gcc-aarch64-linux-gnu` apt package). This is more reliable than QEMU emulation and produces a native arm64 binary.

# v0.57 -- Bootstrap Independence + Vendoring + Compression

## Key Findings

- `./a cc src/cli.a` currently **segfaults** (exit 139). The Rust VM path (`cargo run -- run std/compiler/cgen.a -- src/cli.a`) works and produces **7,942 lines** of C. The bootstrap file must be generated via Rust VM initially.
- miniz **3.1.1** (Feb 2026, MIT license) provides zlib/deflate but **no native gzip support**. We will implement a thin gzip header/trailer wrapper (~30 lines) around raw deflate + CRC-32.
- `flate2` crate will provide Rust VM parity (it uses miniz_oxide internally, a pure-Rust port of miniz).
- No `bootstrap/` directory, `scripts/vendor_check.sh`, or compression code exists anywhere in the codebase yet.

## Part 1: Bootstrap from C

Create `bootstrap/` directory with a pre-generated C file and build script.

- **Generate `bootstrap/cli.c`**: Run `cargo run --quiet -- run std/compiler/cgen.a -- src/cli.a > bootstrap/cli.c`. This is ~7,942 lines, the full CLI including parser + codegen + all stdlib modules.
- **Create `bootstrap/build.sh`**: Standalone script that runs `gcc bootstrap/cli.c c_runtime/runtime.c c_runtime/sqlite3.c -o a -I c_runtime -lm -O2 $STACK_FLAGS $SQLITE_FLAGS`. Detects Darwin for stack flags. No Rust, no cargo.
- **CI verification** in [`.github/workflows/ci.yml`](.github/workflows/ci.yml): After building `./a` via `build.sh`, regenerate the C and diff against the committed `bootstrap/cli.c`. Fail if they diverge. This ensures the committed bootstrap stays in sync.
- **Update [`README.md`](README.md)**: Add a "Bootstrap (no Rust)" section showing `bootstrap/build.sh` as the Rust-free path. Update requirements line.
- **Note**: The `./a cc src/cli.a` segfault is a known limitation (likely stack overflow on the large module tree). The Rust VM path is the canonical generator for now; fixing the native segfault is a separate future task.

## Part 2: Vendor Infrastructure

- **Create `scripts/vendor_check.sh`**: Read `c_runtime/VENDORS.md`, extract expected SHA-256 checksums, compute actual checksums of vendored files, report pass/fail. Exit non-zero on mismatch.
- **Wire into CI**: Add a "Verify vendored checksums" step in [`.github/workflows/ci.yml`](.github/workflows/ci.yml) that runs `scripts/vendor_check.sh`.

## Part 3: Vendor miniz

- **Download** miniz 3.1.1 amalgamation from GitHub releases. The amalgamated form is `miniz.c` + `miniz.h` (two files, ~6k lines combined).
- **Place** as `c_runtime/miniz.c` and `c_runtime/miniz.h`.
- **Update [`c_runtime/VENDORS.md`](c_runtime/VENDORS.md)**: Add miniz entry with version, upstream URL, MIT license, SHA-256 checksums, date vendored.

## Part 4: Compression Builtins (C Runtime)

Four builtins, all string-in -> string-out:

**Declare in [`c_runtime/runtime.h`](c_runtime/runtime.h):**
```c
/* Compression */
AValue a_compress_deflate(AValue data);
AValue a_compress_inflate(AValue data);
AValue a_compress_gzip(AValue data);
AValue a_compress_gunzip(AValue data);
```

**Implement in [`c_runtime/runtime.c`](c_runtime/runtime.c):**
- Add `#include "miniz.h"` at top
- `a_compress_deflate`: Validate TAG_STRING, use `tdefl_compress_mem_to_heap()` for raw deflate, return as `a_string_len()`. Free heap buffer.
- `a_compress_inflate`: Use `tinfl_decompress_mem_to_heap()` for raw inflate, return as `a_string_len()`. Free heap buffer.
- `a_compress_gzip`: Raw deflate the data, then wrap with 10-byte gzip header (magic `\x1f\x8b`, method=8, flags=0, mtime=0, xfl=0, os=0xff), CRC-32 via `mz_crc32()`, and 4-byte original size. Return combined.
- `a_compress_gunzip`: Validate gzip magic bytes, skip header, raw inflate the deflate stream, verify CRC-32 and size. Return decompressed data.
- All error paths return `a_err(a_string("compress.<op>: <message>"))`.

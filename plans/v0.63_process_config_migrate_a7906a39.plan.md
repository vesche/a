# v0.63 -- Process Management + Config

## Context

The roadmap specifies three areas: subprocess orchestration, config loading, and database migrations. From v0.59, we already have `proc.spawn`, `proc.write`, `proc.read_line`, and `proc.kill`. This milestone adds the missing subprocess builtins (`proc.wait`, `proc.is_running`) and two pure "a" stdlib modules (`std/config.a`, `std/migrate.a`).

**Existing infrastructure:**
- `proc.spawn(cmd)` returns int handle, pipes stdin/stdout via fork+exec (C) or `std::process::Command` (Rust)
- `SubProcess` struct in C has `pid`, `stdin_fd`, `stdout_fd`, `active` -- no stderr pipe
- `db.open`, `db.exec`, `db.query`, `db.close` -- bundled SQLite
- `std/toml.a` (parse/stringify), `std/yaml.a` (parse/stringify)
- `env.all()` returns map of all env vars, `env.get`/`env.set`
- `fs.glob(pattern)`, `fs.ls(dir)`, `sort(array)` for file enumeration
- `io.read_file`, `io.write_file`, `str.lines`, `str.ends_with`

## Part 1: New Builtins -- `proc.wait`, `proc.is_running`

### `proc.wait(handle)` -- both runtimes

Waits for the subprocess to exit, reads all remaining stdout, returns a structured result map.

**C runtime** ([runtime.c](c_runtime/runtime.c), ~25 lines): Close `stdin_fd`, read all remaining data from `stdout_fd` into a buffer, `waitpid(pid, &status, 0)`, extract exit code via `WEXITSTATUS`, close `stdout_fd`, mark slot inactive, return `#{ "code": code, "stdout": collected_str }`.

**Rust VM** ([builtins.rs](src/builtins.rs), ~20 lines): Take the child from the slot via `procs[handle].take()`. Call `child.wait_with_output()` which closes stdin and collects stdout. Return `#{ "code": code, "stdout": stdout_string }`.

Note: stderr is not captured (C: not piped; Rust: `Stdio::null()`). The return map includes `"code"` and `"stdout"` only.

### `proc.is_running(handle)` -- both runtimes

**C runtime** (~10 lines): Call `waitpid(pid, &status, WNOHANG)`. If returns 0 -> still running (true). If returns pid -> exited (false, mark inactive). If invalid handle -> false.

**Rust VM** (~10 lines): Call `child.try_wait()`. If `Ok(None)` -> running. If `Ok(Some(_))` -> exited.

### Wiring

- [runtime.h](c_runtime/runtime.h): `AValue a_proc_wait(AValue handle)`, `AValue a_proc_is_running(AValue handle)`
- [cgen.a](std/compiler/cgen.a): `"proc.wait": "a_proc_wait"`, `"proc.is_running": "a_proc_is_running"`
- [builtins.rs](src/builtins.rs) `is_builtin()`: add both
- [checker.rs](src/checker.rs): type signatures -- wait returns `Unknown` (map), is_running returns `Bool`
- [lsp.a](src/lsp.a): completion entries

## Part 2: `std/config.a` (~80 lines)

Pure "a" module. Layered configuration loading using existing parsers and env builtins.

### API

```
config.load(path)              -> map (auto-detect TOML/YAML by extension)
config.env(prefix)             -> map (strip prefix, lowercase keys)
config.dotenv(path)            -> void (parse .env, set via env.set)
config.merge(base, override)   -> map (deep merge, override wins)
config.require(cfg, keys)      -> cfg | Err("missing keys: ...")
```

### Implementation notes

- **`load`**: `io.read_file(path)`, detect format via `str.ends_with(path, ".toml")` vs `.yaml`/`.yml`, call `toml.parse` or `yaml.parse`. Returns parsed map.
- **`env`**: `env.all()` returns full map. Filter keys starting with `prefix + "_"`. Strip prefix, lowercase remaining key. Example: prefix `"APP"`, `APP_PORT=8080` -> `#{"port": "8080"}`.
- **`dotenv`**: `io.read_file(path)`, `str.lines()`, skip blank/comment lines, split on first `=`, `str.trim` key and value, strip optional quotes, `env.set(key, value)`.
- **`merge`**: Walk keys of override map. For each key, if both base and override have map values, recurse. Otherwise override wins. Copy remaining base keys.
- **`require`**: Loop over keys array, check `map.has(cfg, key)`. Collect missing keys. Return `Err` with list or `Ok(cfg)`.

Uses: `use std.toml`, `use std.yaml`

## Part 3: `std/migrate.a` (~90 lines)

Pure "a" module. SQLite migration runner using `db.*` builtins and filesystem ops.

### API

```
migrate.run(db, dir)           -> map with "applied" count
migrate.status(db, dir)        -> [#{"file": str, "applied": bool}]
migrate.create(dir, name)      -> str (path to new migration file)
```

### Implementation notes

- **Bootstrap**: `db.exec(db, "CREATE TABLE IF NOT EXISTS _migrations (name TEXT PRIMARY KEY, applied_at TEXT)")` -- idempotent.
- **`run`**: List SQL files in `dir` via `fs.ls(dir)`, filter `str.ends_with(f, ".sql")`, `sort()` for ordering. For each file, check if already applied via `db.query(db, "SELECT name FROM _migrations WHERE name = ?", [filename])`. If not applied: `io.read_file(path)`, `db.exec(db, sql)`, then `db.exec(db, "INSERT INTO _migrations ...")`.
- **`status`**: Same file listing + query, return structured array.
- **`create`**: Count existing files + 1, zero-pad to 3 digits, write empty file with `io.write_file`.

Convention: files named `001_create_users.sql`, `002_add_email.sql`. Sorted lexicographically.

Uses: `use std.path`, `use std.datetime` (for applied_at timestamps)
